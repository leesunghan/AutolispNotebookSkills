# 코드 검토 및 품질 (AutoLISP Notebook 에디션)

## 개요

노트북 소스 및 AutoLISP 유틸리티의 모든 변경 사양은 배포 또는 머지 전에 정밀 코드 리뷰를 거쳐야 합니다. 코드 리뷰는 정확성(Correctness), 가독성(Readability), 아키텍처(Architecture), CAD 안정성(Safety), 그리고 CAD 최적화 성능(Performance)의 5가지 핵심 축을 기준으로 판단합니다.

---

## 5대 코드 리뷰 핵심 지표 (The Five-Axis Review)

### 1. Correctness (드로잉 로직 정확성)
- AutoLISP 로직이 설계 요구 사항을 예외 오류 없이 충족시키는가?
- 도면 요소를 마우스로 찍어 가져오는 함수(`entsel`, `ssget`) 구문이 빈 클릭(`nil` 반환) 시에 크래시가 나지 않도록 차단 필터를 두고 있는가?
- 캐드 기본 커맨드 호출부들이 다국어 및 해외 캐드 런타임에서도 충돌 없이 돌아가도록 `._` 접두사 규칙을 철저히 준수하는가?
- 임의 회전된 좌표계 평면 상에서도 좌표 기하 평행 연산 공식이 일관성 있게 정밀 좌표를 산출하는가?

### 2. Readability & Simplicity (가독성 및 간결성)
- 노트북 소스가 일관성 있게 마크다운 설명과 AutoLISP 실행 블록을 한 쌍씩 교차하여 읽기 편하게 나열되었는가?
- 축약어나 알 수 없는 작명이 아닌 직관적인 기하 변수명을 부여했는가?
- AutoLISP 고유의 계단식 중첩 괄호 구조가 논리적 깊이 수준에 적절히 맞춰 들여쓰기 처리되었는가?
- 공간 기하 연산이나 벡터 추출 공식들에 대해 이해를 돕는 간결한 주석이 적재적소에 배치되었는가?

### 3. Architecture (아키텍처 및 노트북 설계 규범)
- 학습 편의성을 극대화하기 위해 각 LISP 함수 및 실행 단계가 설명 마크다운과 개별 코드 셀로 충분히 세분화되어 작성되었는가?
- 마크다운 설명 셀의 헤더가 3단계 depth 표준(#, ##, ###)을 엄격히 준수하고 있는가?
- 함수 네임스페이스 오염 방지를 위해 내부 연산용 임시 변수들이 `(defun ... ( / var1 var2 ))` 우측 영역에 확실히 기입되어 지역 변수화(localization) 되었는가?
- ActiveX 개체 라이브러리 참조 함수 호출 직전에 `(vl-load-com)`을 선행 로드하도록 확인 구문이 적용되어 있는가?

### 4. CAD Safety (사용자 도면 환경 보존)
- 사용자가 기존에 조율해 둔 기본 캐드 옵션 변수(`OSMODE`, `CMDECHO` 등)를 수정하기 전 임시 백업을 확보하고, 정상 구동 성공 시와 비상 예외 탈출(`*error*` 핸들러 호출 시) 시에 모두 본래 상태로 완벽히 리셋시켜 주는가?
- 실제 도면에 선을 치는 연산 그리기 과정에서는 좌표가 왜곡되지 않도록 OSNAP을 일시 비활성화(`OSMODE` 0) 처리하고, 마우스로 지점을 찍어야 하는 사용자 응답 시점에는 OSNAP을 도로 켜 주는가?
- 세분화된 흐름 전체의 실행 취소를 유기적으로 관리하기 위해 첫 단계 셀과 마지막 완료 단계 셀에 Undo group 시작과 끝이 짝을 이루고 있는가?

### 5. CAD Performance (실행 속도 최적화)
- 대량의 자동 그리기 작업 도중에 화면이 번쩍이고 깜빡이며 내부 부하를 가중시키는 것을 차단하기 위해 명령 콘솔 반응(`CMDECHO` 0) 제어가 선행 처리되는가?
- 선택 세트(`ssget`)로 요소를 추출할 때 일단 다 퍼 올린 뒤 루프를 도는 형태를 탈피하여, 쿼리 구문 상에 명확한 DXF 필터 속성을 걸고 필요한 개체만 낚아 올리는가?
- 대량의 부재 생성 및 변경 반복 순회 작업에서 속도가 떨어지는 내장 커맨드 그리기 방식 대신 ActiveX 그리기 전용 메소드들을 적극적으로 배치했는가?

## 코드 리뷰 체크리스트 (Review Checklist)

```markdown
## Review: [노트북 / AutoLISP 기능 명칭]

### Context (설계 요구 사항 파악)
- [ ] 개발하려는 CAD 자동화 기능의 제도 목적과 요구 정밀도를 인지하고 있다.

### Correctness (기능 작동 확인)
- [ ] 구현한 AutoLISP 수식과 좌표 산출 로직이 도면 규격에 부합한다.
- [ ] 엔티티 지정 실패(`nil` 반환) 사태에 대한 방어 로직이 적용되어 있다.
- [ ] 캐드 내장 커맨드가 국제 표준 접두사인 `._` 형태로 매핑되어 있다.

### Readability (가독성 검수)
- [ ] 설명(Markdown)과 실행(AutoLISP) 셀의 정돈된 교차 배치가 지켜지고 있다.
- [ ] AutoLISP 변수명 및 커스텀 함수들이 의미론적으로 명확하게 지어졌다.
- [ ] 괄호 깊이 수준이 알맞게 들여쓰기되어 읽기 쉽.

### Architecture & Safety (아키텍처 및 도면층 안전성 검수)
- [ ] 개별 코드 셀에 불필요한 `(progn ...)` 래핑이 배제되어 있다.
- [ ] LISP 함수 및 단계 단위로 설명 마크다운과 코드 셀이 세분화되어 있고, 마크다운 3 depth 구조가 잘 구현되어 있다.
- [ ] 소스 내부 연산 변수들이 로컬화 구문 `(defun ... ( / 변수들 ))`에 완전히 등록되어 있다.
- [ ] 세분화된 셀 흐름 전반에 걸쳐 Undo group 시작과 끝이 짝을 이루어 유기적으로 연계되어 있다.
- [ ] 사용자의 `OSMODE` 및 `CMDECHO` 원본 백업이 에러 탈출(`*error*`) 시에도 무조건 리셋되도록 보장되어 있다.


### Performance (최적화 성능 검수)
- [ ] 자동 드로잉 실행 도중 화면 및 콘솔 반사 억제(`CMDECHO` 0)가 가동된다.
- [ ] ssget 선택 세트 조회 시 필터 지정을 명시하여 메모리 낭비를 줄였다.
- [ ] 대용량 드로잉 루프 내부 성능 확보를 위해 ActiveX 방식 그리기를 배치했다.

### Verdict (최종 코드 검수 승인 판정)
- [ ] **Approve (승인)** — 안전 규격을 충족하여 도면 적용 준비 완료
- [ ] **Request changes (반려)** — 안정성 결함이나 예외 원복 누락이 식별되어 수정 보완 필요
```
