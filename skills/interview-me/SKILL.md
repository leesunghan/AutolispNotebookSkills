# 나를 인터뷰해줘 (AutoLISP Notebook 에디션)

## 개요

CAD 자동화 영역에서 사용자가 구상하는 두루뭉술한 요건("벽을 그리는 오프셋 기능", "교차부 자동 정리 기능")을 구체적 수치 사양 없이 받아들면 오작동의 온상이 됩니다. 만일 타겟 캐드 기종(AutoCAD vs BricsCAD), 도면 제도 단위(mm / inches), 회사 표준 도면층 색상/이름 체계, 혹은 드로잉 시점의 OSNAP 제어 범위 등을 설계 전에 명확히 짚고 넘어가지 않으면, 엉뚱한 위치에 선이 꼬여서 그려지거나 CAD 런타임 크래시를 유발하게 됩니다.

이 스킬은 본격적인 스펙 작성이나 코딩에 앞서 이러한 틈새 요건들을 확실히 수집하도록 유도합니다. 에이전트가 당연하다고 넘겨짚은 가정을 가설 형태로 하나씩 제시하며 질문을 던져 캐드 환경 설정과 기하 수학 공식을 빈틈없이 구체화하는 단계입니다.

## 사용 시점

- 작업 목표 중 다음의 필수 파라미터가 누락되었을 때: **타겟 캐드 엔진 종류** (AutoCAD / BricsCAD), **도면 유닛 단위** (mm / inches 등), **도면층 표준** (AIA 표준, 한국건설공사 표준 KCS 등), **기하 제어 값** (벽체 두께 규격, 오프셋 간격 수치).
- 모호한 문장("도면 튜토리얼을 작성해 달라")을 세부적인 노트북의 순차 교차 셀 구성(Cell-by-cell)안으로 세부 해부하여 정리할 필요가 있을 때.
- 머릿속으로 사용자 도면 환경 상태 변수 값들을 확인 없이 임의로 설정하여 구상하고 있을 때.

**사용하지 않는 시점:** AutoLISP 소스의 단일 라인 수학 공식 보정이나 오타 조율 등 극도로 단순한 경우, 혹은 이미 요구 사양서에 유닛/레이어/기하 치수 규격이 빈틈없이 완성되어 주어진 경우.

## 질답 구체화 단계 (The Process)

### Step 1: 가정 수립 및 자가 신뢰도 파악 (Hypothesize)
질문을 던지기에 앞서, 에이전트가 파악하고 있는 요구 요건을 명확히 요약하여 예상되는 작업의 자가 신뢰도 수치(0-100%)와 함께 선제 제시하십시오:
```
[자가 신뢰 가설]: BricsCAD V24 환경에서 미국 건축 표준 AIA 규격 레이어를 기반으로 구동되는 벽체 작도용 AutoLISP Notebook 작성을 의도하시는 것으로 파악했습니다.
신뢰 수준: ~40% — 미확인 요건: 실제 도면 제도 단위(mm인지 인치인지 여부), 구체적인 벽 두께 치수, 모서리 교차점 모따기 적용 여부.
```

### Step 2: 한 번에 하나씩 가설이 동반된 질문 던지기 (Ask one question at a time)
형식:
```
Q: 대상 도면층 표준과 제도 단위는 어떻게 설정할까요? (예: 한국건설공사 KCS 표준 레이어 체계 및 밀리미터 단위를 적용할까요?)
GUESS (가설): 현재 사용자의 이전 작업 이력 분석 결과 KCS 표준에 기반한 벽체선(A-WALL-CTR, S-WALL-CONC 등)을 mm 단위(1유닛 = 1mm)로 설계한 경우가 가장 빈번했으므로, 이와 동일한 제도로 구상하고 계실 것으로 추측합니다.
```
*주의: 여러 개 질문을 한 번에 목록으로 쏟아내지 말고, 반드시 사용자가 한 질문에 대한 대답을 해 주면 그것을 바탕으로 다음 단계 질문을 이어서 던지십시오. 그래야 정확한 요구사항의 합의점에 도달할 수 있습니다.*

### Step 3: 관습적 응답이나 무조건적 예외 경계 (want vs should want)
사용자가 "그냥 대충 캐드 기본값으로 진행해 달라"며 정밀 검토 없이 관습적인 답변을 하는 경우, 대충 넘겨짚지 말고 더 뾰족하게 캐물어 세팅을 특정하십시오:
> *"특정 회사 규범이나 복잡한 표준을 적용할 필요가 전혀 없다면, 지금 열어두신 작업 도면 화면 상에서 실제로 구현될 벽체의 정확한 두께 치수(예: 콘크리트 옹벽 중심선 기준 좌우 100mm씩 대칭 평행 오프셋하여 총 200mm 벽체 작성)와 도면층 색상은 어떤 임의 값으로 고정하여 보여드릴까요?"*

### Step 4: 요구 요건 최종 환원 및 확인 (Restate Intent)
신뢰도가 90% 이상 확보되면, 사용자가 확인하기 편하게 최종 합의 요건을 환원 보고하십시오:
```
다음과 같은 CAD 자동화 사양으로 구현 요건이 조율 완료되었습니다:
- 최종 산출물: [예: 벽체 및 중심선 그리기를 제어하는 AutoLISP Notebook (.lspnb) 파일]
- 타겟 CAD 엔진: [AutoCAD / BricsCAD]
- 제도 단위: [Millimeters (1 unit = 1mm)]
- 레이어 표준: [KCS 표준 준수 (벽체선은 3번 초록색 S-WALL-CONC, 마감선은 4번 하늘색 A-WALL-FINS)]
- 기하 작도 규칙: [총 두께 200mm 벽체선, 대칭 100mm씩 평행 오프셋 제도, 코너부 깎기 마이터 정리 포함]
- 범위 외 항목: [문/창호 전용 개구부 작성 로직, 도면 치수선 및 해치 패턴 작성은 이번 작업 범위에서 전면 제외]

위 조율 내역이 최종 맞는지 "예" 혹은 "보완 요청"을 입력해 주십시오.
```

### Step 5: 최종 승인 동의 득하기 (Confirm)
사용자의 명확하고 적극적인 "예(Yes)" 혹은 확인 피드백을 확보하기 전에는 결코 임의 스펙이나 태스크 분할 단계로 도약하여 구현을 시도해서는 안 됩니다.
