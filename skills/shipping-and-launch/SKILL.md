# 배포 및 출시 (AutoLISP Notebook 에디션)

## 개요

작성한 AutoLISP Notebook(`.lspnb`) 파일을 사용자에게 자신 있게 전달하십시오. 노트북 파일이 VS Code 상에서 경고나 지연 없이 한 번에 싹 열리고, 실행 버튼을 순서대로 눌렀을 때 사용자의 실제 CAD 도면 상에 어떠한 에러도 남기지 않으며 안정적으로 제도 자동화를 완수하는 상태임을 검증하고 최종 릴리즈하는 절차입니다.

## 사용 시점

- 새로 조립한 AutoLISP Notebook(`.lspnb`) 학습 및 자동화 튜토리얼 파일을 릴리즈할 때.
- CAD 실무 제도용 AutoLISP 유틸리티 도구를 완성하여 배포용 문서로 가공할 때.
- 캐드 자동화 교육 자료용 노트북 예제를 공유하고자 준비할 때.

## 배포 전 필수 검수 체크리스트 (The Pre-Launch Checklist)

사용자 환경으로 파일을 전달하기 전에 아래 항목들을 누락 없이 이행했는지 확인하십시오:

### 1. JSON Format & Extension Loading (JSON 무결성 및 확장 도구 로딩)
- [ ] 대상 노트북 파일이 `projects/<프로젝트명>/` 하위에 올바르게 정돈되어 배포될 상태인가?
- [ ] 완성된 `.lspnb` 파일이 문법 오류가 없는 정밀한 JSON 포맷을 유지하는가?
- [ ] AutoLISP 코드 라인들을 담은 JSON 소스 배열 안에서 특수기호나 따옴표(예: `\"`, `\\`, `\r\n`)의 이스케이프 처리가 완벽히 준수되는가?
- [ ] VS Code 상에서 AutoLISP Notebook 확장을 통해 열었을 때, 구문 깨짐이나 유효하지 않은 셀 경고 없이 매끄럽게 렌더링되어 파싱되는가?

### 2. Cell Alternation & Readability (설명과 코드의 교차 및 가독성)
- [ ] 전체 노트북 구성이 다음 순서에 따라 어긋남 없이 정렬되어 있는가? **마크다운 설명 셀 ──→ AutoLISP 코드 셀 ──→ 마크다운 설명 셀 ──→ AutoLISP 코드 셀**.
- [ ] 마크다운 지침이 복잡한 조작 단계를 알기 쉽게 나누어 친절한 텍스트로 풀어 설명하고 있는가?
- [ ] AutoLISP 주석이 기하 벡터 계산 방식이나 평행 오프셋 산식에 대해 명확한 설명을 담고 있는가?

### 3. AutoLISP Cell Safety (AutoLISP 셀 코드의 안전 규격 준수)
- [ ] 개별 코드 셀 전체가 예외 없이 단일 `(progn ...)` 블록 구조로 완전히 감싸져 있는가?
- [ ] 모든 개별 코드 셀 내부의 작업 실행 범위가 실행 취소 그룹화(`_.undo _group` / `_.undo _end`) 속에 포함되어 단 한 번의 실행 취소(`Ctrl+Z`)로 초기화될 수 있는가?
- [ ] 캐드 환경 제어 설정 변수(`OSMODE`, `CMDECHO`)가 실행 전 임시 백업되고, 정상 종료 시 정상 환원되는가?
- [ ] 임의 중도 취소(Cancel 또는 ESC 작동)가 발생했을 때 백업 변수들을 무조건 원상 복귀해 주는 로컬 `*error*` 비상 핸들러 선언과 복원 처리가 누락 없이 탑재되었는가?
- [ ] 모든 변수가 `(defun ... ( / local_vars ))` 우측에 명시되어 전역 환경 네임스페이스 오염 우려를 지워냈는가?

### 4. CAD Compatibility & Performance (캐드 기종 호환성 및 화면 반응성)
- [ ] 모든 오토캐드 내장 명령어 호출 시 다국어 및 타 기종 호환성을 위해 `._` 접두사를 붙여 주었는가?
- [ ] 오토캐드(AutoCAD)뿐만 아니라 브릭스캐드(BricsCAD) 환경에서도 충돌 없이 동일하게 작도 연산이 돌아가는가?
- [ ] 대용량 작도 반복 동작 시 불필요하게 콘솔 화면을 밀어내어 지연을 가중시키는 병목 현상이 없도록 명령 에코 억제(`CMDECHO` 0)가 동작하는가?
- [ ] 도면 메모리 정리를 위해 할당한 선택 세트와 사용이 끝난 ActiveX 개체들을 메모리에서 회수/정리하는 구문이 적재적소에 들어갔는가?

### 5. Verification Assertions (성능 검증용 어설션 구성)
- [ ] 사용자가 작업을 제대로 끝마쳤는지 스스로 자가 진단할 수 있는 검증용 어설션 코드 셀이 노트북 최하단부에 명확히 조립되어 있는가?
- [ ] 검증 코드 작동 시 현재 도면 상의 생성된 엔티티 형상과 레이어의 속성을 정확하게 AutoLISP로 질의하여 합격 여부를 판별해 주는가?

## 최종 배포 필드 테스트 요령 (Post-Shipping Spot Check)

배포 직전 최종 본에 대해 아래 수동 필드 검사를 반드시 행하십시오:
1. 아무것도 그려지지 않은 완전한 빈 캐드 템플릿 도면 파일을 엽니다.
2. 작성 완료한 `.lspnb` 파일을 열어 제일 첫 셀부터 순서대로 버튼을 클릭하며 한 셀씩 돌려봅니다.
3. 드로잉 결과로 그려진 실물 벽체 좌표 및 치수가 스펙 설계와 밀리미터 단위까지 완전히 일치하는지 치수선 등으로 실측 대조합니다.
4. 그 직후 키보드에서 `Ctrl+Z`를 한 번만 눌러 방금 그린 모든 요소가 완전히 도면에서 회수되어 빈 도면 상태로 깨끗하게 회귀하는지 체크합니다.
5. 중간에 명령 취소(ESC)를 강제로 행한 뒤에도 마우스 스냅 설정(`OSMODE`)과 명령 줄 에코 모드(`CMDECHO`)가 본인의 원본 세팅과 완전히 일치하게 복구되었는지 조율 값을 확인합니다.
