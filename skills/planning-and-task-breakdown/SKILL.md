# 계획 수립 및 작업 분할 (AutoLISP Notebook 에디션)

## 개요

노트북 작성 및 AutoLISP 코딩 작업을 구체적으로 동작 검증이 가능한 미세 단계(태스크)로 분할하십시오. 꼼꼼한 작업 계획 수립만이 CAD 프로세스 동결이나 뒤엉킨 난해한 코드를 방지하는 척도입니다. 하나의 태스크는 개발 에이전트가 단일 작업 주기 내에 독립적으로 완성하고 검증할 수 있을 만큼 가볍고 작아야 합니다.

## 사용 시점

- 확정된 스펙 문서를 바탕으로 노트북 파일 내의 셀(Cell)들을 순차적으로 구성하고자 할 때.
- 해결하려는 AutoLISP 기하 좌표 공식(예: 벽 교차 구간의 벽선 모따기 및 자동 트림 로직)이 한 번에 짜기 복잡하게 얽혀 있을 때.
- 사용하려는 CAD 명령어 목록과 ActiveX API의 흐름 및 우선순위를 정립하고자 할 때.

**사용 불가능한 시점:** 한두 줄짜리 단순 수정이나, 이미 요구사항 정의서에 세부 셀 기획서가 상세히 제공된 경우.

---

## 계획 수립 단계 (The Planning Process)

### Step 1: 계획 모드 진입 (Enter Plan Mode)
노트북 파일을 만들거나 AutoLISP 문법을 치기 전에 반드시 정독 모드로 상황을 분석하십시오:
- 요구하는 표준 도면층 이름, 선종류, 치수 데이터 등의 도면 상세 설정을 정확히 확인합니다.
- 좌표 변환 과정에서 어떤 기하 벡터 연산과 방향 전환(Normal vector)이 필요한지 수식으로 미리 정리합니다.
- 프로젝트 내에 사전에 구축된 공통 AutoLISP 헬퍼 함수가 있는지 살핍니다.

**이 과정에서는 결코 소스 코드를 타이핑하지 마십시오.** 계획 단계의 핵심 산출물은 "계획 문서"이지 소스 코드가 아닙니다.

### Step 2: 의존성 그래프 도출 (Identify the Dependency Graph)
AutoLISP Notebook을 조립할 때 발생하는 전형적인 하부 기술적 의존성 그래프는 다음과 같이 흘러갑니다:

```
도면 작도 환경 세팅 (도면층 구성, 텍스트 스타일 지정, CMDECHO 제어)
     │
     ├── 사용자 점 입력 및 타겟 엔티티 선택 (getpoint, entsel)
     │       │
     │       ├── 기하학적 평행/수직 벡터 연산 (오프셋 거리 및 투영 좌표 도출)
     │       │       │
     │       │       ├── 도면 요소 그리기 실행 (선, 폴리선, 원 등 생성)
     │       │       │       │
     │       │       │       └── 모서리 필렛 정리 및 교차점 트림 작업
     │       │
     │       └── 사용자 작업 중도 탈출 시 환경 변수 회복 조치 (*error* 복구)
     │
     └── 도면 무결성 입증 검증 어설션 코드 실행 (tblsearch, ssget 데이터 조회)
```

### Step 3: 세로 방향으로 슬라이싱하기 (Vertical Notebook Slicing)
노트북의 셀 쌍(Markdown + AutoLISP Code)을 조각 단위로 점진적으로 확장해가며 코딩해야 합니다. 노트북을 만들 때 설명 마크다운 셀들만 한꺼번에 다 써놓은 다음, AutoLISP 코드를 나중에 한 번에 쏟아 넣는 것은 잘못된 개발 방식입니다.

**Bad (가로 방향 조립 - .lsp 및 .dcl 파일을 별도로 다루는 방식):**
```
Task 1: 전체 노트북 시나리오의 마크다운 셀들을 싹 다 타이핑한다.
Task 2: AutoLISP 및 DCL 코드 전체를 별도의 외부 .lsp 및 .dcl 파일에 따로 다 짠다.
Task 3: 작성한 코드들을 대충 복사해서 노트북 JSON의 각 코드 셀 소스 안에 쪼개어 밀어넣는다.
```

**Good (세로 방향 셀 단위 슬라이싱):**
```
Task 1: Cell 1(안내 설명 마크다운) 및 Cell 2(AutoLISP: 도면층 확인 및 생성, CMDECHO 억제, Undo 그룹 시작 시점 마킹)를 세트로 작성하고 실제로 CAD에 올려 작동을 검증한다.
Task 2: Cell 3(마우스 점 클릭 설명) 및 Cell 4(AutoLISP: 객체 포인트 지정, OSNAP 복원, 두 점 사이의 법선 벡터 연산 및 OSNAP을 끄고 벽체선 오프셋 작도)를 완성하고 정상 선그리기를 검증한다.
Task 3: Cell 5(벽체 모서리 정리 설명) 및 Cell 6(AutoLISP: 추출한 교차점 좌표들에 대해 Fillet 커맨드 적용, 비상 예외 방어 기능 활성화)를 얹고 도면이 깔끔하게 다듬어지는지 검증한다.
Task 4: Cell 7(검증 안내 설명) 및 Cell 8(AutoLISP: 작성된 폴리선과 레이어가 정확한지 데이터베이스 쿼리를 던져 PASS/FAIL 판별)를 구현하여 도면 무결성을 검증한다.
```

### Step 4: 세부 태스크 명세 작성 (Write Tasks)
작성할 개별 태스크 카드는 반드시 다음 구조를 지켜 문서화하십시오:

```markdown
## Task [N]: [노트북 셀 지정 기능 명칭]

**Description (작업 상세):** 이 셀 세트가 수행할 동작을 요약 서술합니다 (예: 사용자에게 벽체 중심선을 선택하도록 요청하고, 양방향으로 100mm 오프셋하여 벽체 외형선 작성).

**Target File (대상 파일):** `projects/<프로젝트명>/<노트북명>.lspnb`

**Acceptance criteria (인수 조건):**
- [ ] 마크다운 설명 셀이 사용자가 마우스 조작을 쉽게 따라 할 수 있도록 정리되어 있다.
- [ ] AutoLISP 코드 셀 실행 시 AutoCAD 및 BricsCAD 상에서 예외 에러 없이 매끄럽게 돌아간다.
- [ ] 사용자 취소 시에도 캐드의 OSNAP 환경이 무너지지 않고 정상 복원된다.

**Verification (검증 요령):**
- [ ] 작성한 AutoLISP 셀이 CAD 환경에서 경고 없이 로드되어 실행된다.
- [ ] CAD 데이터베이스 검색 `(ssget "_X" '((8 . "S-WALL-CONC")))` 호출 시 정확히 양 방향 평행 오프셋된 선 2개가 잡힌다.

**Dependencies (선행 의존 태스크):** [예: Task 1 (도면층 초기화 준비 셀)]

**Estimated scope (추정 작업량):** [Small: 1개 셀 세트 | Medium: 2개 이상의 셀 세트]
```

### Step 5: 순서 배치 및 중간 검문소 (Order and Checkpoint)
아래 흐름을 검토하여 최종 계획의 조율 상태를 점검하십시오:
1. 도면층 생성이나 스타일 초기화 등 기초가 되는 환경 선언 셀이 무조건 최우선 배치되어야 합니다.
2. 각 코드 셀은 독립적으로 바로 실행이 가능하거나, 이전 단계의 코드 셀 실행 결과로 올라온 메모리 상의 함수/변수에 안전하게 의존해야 합니다.
3. 중심선 그리기 단계 완료, 벽 그리기 단계 완료, 교차점 트리밍 단계 완료 시점에 각각 중간 확인 검문소(Checkpoint)를 마련해 작동을 끊어서 입증하도록 배치합니다.

## 구현 계획서 템플릿 (Plan Document Template)

```markdown
# Notebook Implementation Plan: [기능 명칭]

## Overview (개발 개요)
[이 노트북 튜토리얼 자동화 과정의 학습 목표 및 구현 내용 설명]

## Architecture & CAD Decisions (캐드 설계 의사결정)
- 타겟 CAD 엔진: [AutoCAD/BricsCAD]
- 도면층 명명 표준: [KCS 표준 기반 A-WALL-CTR, S-WALL-CONC 등 적용]
- OSNAP & Undo 관리 전략: [시스템 변수의 임시 백업 구조 및 실행 취소 범위 설계 방식]

## Cell Task List (셀별 태스크 목록)

### Phase 1: Setup & Environment (도면 설계 준비 환경 세팅)
- [ ] Task 1: Cell 1 & 2 작성 (소개 마크다운 및 도면층 자동 빌드 기능)
- [ ] Task 2: Cell 3 & 4 작성 (사용자 마우스 선 지정 및 기하 투영 좌표 계산)

### Checkpoint: Environment Setup (1단계 검문소)
- [ ] 표준 레이어가 로드되어 활성화되고, 사용자의 중심선 벡터 좌표 추출 연산이 오류 없이 작동한다.

### Phase 2: Drawing & Detailing (벽체 드로잉 및 상세 가공)
- [ ] Task 3: Cell 5 & 6 작성 (계산 좌표 기준 벽체선 그리기 기능)
- [ ] Task 4: Cell 7 & 8 작성 (모서리 벽 교차 구간 오차 정리 기능)

### Checkpoint: Completed Geometry (2단계 검문소)
- [ ] 벽체 선그리기와 교차선 트림 정리가 깔끔하게 완료되며, 비상 정지 시에도 캐드의 원본 시스템 변수가 정상 복원된다.

### Phase 3: Verification (데이터베이스 무결성 검증)
- [ ] Task 5: Cell 9 작성 (데이터베이스 개체 쿼리 및 좌표 일치 여부 판별 검증 셀)

## Risks and Mitigations (잠재적 리스크 및 예방책)
- 그리기 동작 도중 사용자가 설정해 둔 스냅 지점으로 선이 끌려 들어가 좌표가 어긋나는 경우 -> 예방책: 그리기 명령 수행 전 반드시 OSMODE를 0으로 통제하고 완료 직후 원래 백업한 값으로 되살린다.
```
