# CAD 및 AutoLISP 성능 체크리스트

가볍고 빠르며 반응성이 우수한 AutoLISP 루틴 및 노트북 셀을 작성하기 위한 성능 체크리스트입니다. 이 파일은 `cad-performance-auditor` 에이전트 페르소나와 함께 사용하십시오.

## 성능 목표치

- **단순 유틸리티 및 조회 작업**: 실행 시간 ≤ 100ms.
- **대량 도면 객체 생성 (1,000개 이상)**: 실행 시간 ≤ 2.0초.
- **CAD 프리징 방지**: 긴 루프가 실행될 때 운영체제에서 "응답 없음"을 유발하여 시스템이 멈추는 현상이 없어야 합니다.

## 콘솔 및 뷰포트 업데이트 억제

콘솔에 텍스트를 출력하거나 도면 화면을 계속 새로 고치는 작업은 CAD 엔진 성능 저하의 주된 요인입니다.

- [ ] **CMDECHO 제어**: 실행 중에 불필요한 명령행 입출력이 로깅되는 현상을 막기 위해 LISP를 시작하기 전에 `(setvar "CMDECHO" 0)`을 선언하고 마칠 때 원복합니다.
- [ ] **NOMUTT 제어**: 도면 그리기 작업 중 명령행 메시지가 출력되지 않도록 `(setvar "NOMUTT" 1)`을 설정합니다.
- [ ] **화면 재생성 억제**: 대량의 그리기 루프 안에서 불필요하게 `(command "._regen")`을 실행하거나 뷰포트를 강제로 업데이트하지 마십시오. 필요한 경우 그리기 작업 중에 특정 레이어를 얼렸다(Freeze)가 작업이 완료되면 녹이는(Thaw) 방식으로 리프레시 횟수를 줄입니다.
- [ ] **단일 Undo 마크 활용**: 복잡한 제도 루틴 전체를 `vla-StartUndoMark`와 `vla-EndUndoMark`로 감싸십시오. 이는 트랜잭션을 묶어주고 개별 객체가 생성될 때마다 매번 화면이 깜빡이거나 새로 그려지는 현상을 완화합니다.

## 데이터베이스 및 선택 세트 쿼리

비효율적인 데이터베이스 탐색은 대용량 도면 파일에서 O(N^2)의 성능 저하를 야기합니다.

- [ ] **DXF 필터 쿼리**: `ssget`으로 도면 객체를 검색할 때 인자가 없는 무조건적인 전체 스캔을 지양하고, 필터 리스트를 추가하여 캐드 엔진 자체에서 필터링하게 하십시오.
  - **나쁜 예**: `(ssget "_X")`로 다 가져와서 LISP 루프로 레이어를 일일이 검사.
  - **좋은 예**: `(ssget "_X" '((0 . "LINE") (8 . "A-WALL")))`로 사전에 필터링된 결과만 가져옴.
- [ ] **선택 세트 즉시 메모리 해제**: 조회가 끝난 selection set 변수는 `(setq ss nil)`을 선언하여 사용 즉시 힙 메모리에서 지워야 자원 누수를 막을 수 있습니다.
- [ ] **직접 데이터 참조**: 엔티티의 특정 속성을 조회할 때 명령행 도구로 객체를 변환/확인하지 말고, DXF 연관 리스트 `(assoc 10 (entget ent))` 또는 VLA 속성을 조회하여 메모리에서 직접 값을 가져오십시오.

## ActiveX API 가속

ActiveX direct memory 호출은 문자열 파서와 명령 인터프리터를 거치지 않아 도면 그리기 속도를 10배에서 100배까지 빠르게 만들어 줍니다.

- [ ] **ActiveX를 통한 그리기**: 다량의 반복 루프 내부에서는 `(command "._line" ...)`을 쓰지 않고 `(vla-AddLine ...)` 또는 `(vla-AddLightWeightPolyline ...)` 함수를 사용합니다.
- [ ] **속성 직접 설정**: 레이어나 색상을 변경할 때 `(command "._change" ...)`나 `entmod` 대신, VLA 객체의 속성 값 수정 메서드 `(vla-put-Layer ...)`와 `(vla-put-Color ...)`를 활용합니다.
- [ ] **COM 객체 한 번만 로드**: LISP 스크립트 상단에 `(vl-load-com)`을 선언하고, active document 객체 등은 루프 외부에서 미리 바인딩한 후 캐싱하여 루프 안에서 재사용합니다.
- [ ] **COM 객체 해제**: 사용 완료한 VLA 객체들은 `(vlax-release-object)`를 명시적으로 호출하거나 `nil`로 덮어씌워(특히 BricsCAD의 경우) 가비지 컬렉터가 원활하게 처리할 수 있도록 돕습니다.

## 루프 및 기하학 연산 최적화

- [ ] **삼각함수/수학 연산 캐싱**: 반복 루프 내에서 매번 계산할 필요가 없는 삼각함수 `(sin ang)`, `(cos ang)`이나 단순 좌표 변환 연산 등은 외부 변수에 미리 계산해 두고 재활용합니다.
- [ ] **중복 엔티티 정보 쿼리 방지**: 반복 호출되는 객체의 기하 정보는 변수에 리스트 구조로 담아 재조회를 최소화합니다.
- [ ] **재귀 대신 반복문 활용**: LISP 특유의 깊은 재귀 호출 대신 `repeat`이나 `while`을 활용하여 메모리 오버플로우나 스택 오버플로우를 예방합니다.
- [ ] **REGEN 1회 실행**: 도면 리젠 연산 `(vla-Regen ...)` 또는 `(command "._regen")`은 반드시 전체 드로잉 루틴의 최종 종료 단계에서 단 1회만 호출되어야 합니다.

## 자주 발생하는 성능 안티패턴

| 안티패턴 | 미치는 영향 | 대안 및 해결책 |
|---|---|---|
| 루프 내부의 Command 그리기 | 파서 처리로 인해 극심한 속도 저하 발생 | ActiveX `vla-Add...` 또는 `entmake` 사용 |
| 필터 없는 `(ssget "_X")` | 전체 DWG 파일 데이터베이스를 긁어 프로그램 멈춤 발생 | 레이어, 객체 타입 등 DXF 필터 그룹 지정 |
| 루프 내부의 Regen 호출 | 객체가 그려질 때마다 전체 화면 다시 로드 | 최종 종료 단계에서 1회만 재생성 실행 |
| 전역 변수 선택 세트 방치 | 메모리 누수 발생 및 다른 작업 영역 오염 | 선택 세트 변수를 함수 매개변수 뒤 `/` 영역에 선언하여 지역화 |
| 반복적인 삼각함수 연산 | 쓸데없이 CPU 부하 가중 | 루프 외부에서 변수에 선언한 후 참조 |
