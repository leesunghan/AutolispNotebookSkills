## 안전 및 위협 모델링

LISP 스크립트를 로드하거나 노트북 셀을 작동시키기 전에 다음과 같은 위험 요소들을 면밀히 점검하십시오:
- **환경 상태**: 만약 사용자가 동작 도중에 `Esc`를 눌러 강제 중단하면 CAD 환경 변수가 훼손된 채로 방치되는가?
- **도면 상태**: 선택 세트에서 도면 요소가 0개 검색되는 등의 비정상 상태일 때 에러가 나면서 멈추는가?
- **연산 오류**: 나누기 0 연산, 일치하는 점 좌표 연산, 음수 길이 등 수학적 기하 오류가 프로그램 크래시를 유발하는가?

## 변수 스코프 및 네임스페이스 안전

- [ ] **임시 변수 지역화**: 작성한 함수 내의 모든 임시 변수는 `(defun ...)` 구문의 슬래시 `/` 뒤편에 확실히 선언해야 합니다. 지역화되지 않은 임시 변수는 전역 변수 영역으로 누출되어, 동일 세션 내의 다른 LISP 실행을 오염시킵니다.
  - **나쁜 예**: `(defun c:draw-box () (setq p1 '(0 0) p2 '(10 10)) ...)`
  - **좋은 예**: `(defun c:draw-box (/ p1 p2) (setq p1 '(0 0) p2 '(10 10)) ...)`
- [ ] **고유 접두사 네임스페이스**: 전역 헬퍼 함수나 서브루틴은 AutoCAD/BricsCAD 내부 명령어나 타사 서브루틴과 충돌하지 않도록 고유한 접두사(예: `myproject:`)를 붙이십시오.

## 시스템 변수 및 상태 복구

- [ ] **시스템 변수 캐싱**: 작업 영역 내부에서 임의로 수정하려는 모든 환경 변수(`OSMODE`, `CMDECHO`, `NOMUTT`, `ANGBASE`, `CLAYER` 등)는 실행 전에 원래 값을 로컬 변수에 캐싱해야 합니다.
- [ ] **환경 설정 복원**: 정상적인 함수 종료 직전에 백업해 둔 변수들을 복원합니다.
- [ ] **실행 취소(Undo) 마크 정합성**: `vla-StartUndoMark`를 활성화했다면 반드시 이에 상응하는 `vla-EndUndoMark`가 짝을 맞추어 닫히도록 감싸십시오. 언두 영역을 닫지 않으면 캐드의 언두 히스토리 스택 전체가 오염되거나 손상될 수 있습니다.

## 오류 처리 및 예외 가로채기

LISP는 CAD의 주 스레드(main UI thread)에서 작동합니다. 정상 종료가 안 되었을 때의 예외 처리가 부실하면 사용자의 CAD 환경이 비정상적으로 왜곡된 채 지속됩니다.

- [ ] **로컬 에러 핸들러 구성**: 진입 함수 내부에 로컬 `*error*` 함수를 정의하여 ESC 키 입력이나 오류로 비정상 종료 시 시스템 변수를 복원하고, 실행 취소 마크를 마감하며, 콘솔에 에러 메시지를 출력할 수 있도록 구성하십시오.

```lisp
(defun my-draw-cmd (/ old-osmode old-cmdecho *error*)
  ;; 로컬 에러 함수 정의
  (setq *error* (lambda (msg)
                  (if old-osmode (setvar "OSMODE" old-osmode))
                  (if old-cmdecho (setvar "CMDECHO" old-cmdecho))
                  (if (and doc (= 1 (vla-get-ActiveUndoMark doc)))
                      (vla-EndUndoMark doc)
                  )
                  (princ (strcat "\n오류: " msg))
                  (princ)
                )
  )
  
  ;; 시스템 변수 백업
  (setq old-osmode (getvar "OSMODE")
        old-cmdecho (getvar "CMDECHO")
  )
  
  ;; 실행 취소 마크 시작
  (vla-StartUndoMark doc)
  
  ;; 메인 제도 로직이 위치하는 곳...
  
  ;; 복원 및 실행 취소 마크 종료
  (vla-EndUndoMark doc)
  (setvar "OSMODE" old-osmode)
  (setvar "CMDECHO" old-cmdecho)
  (princ)
)
```

## 무한 루프 및 크래시 예방

- [ ] **선택 세트 루프 바운더리 검사**: 선택 세트의 요소를 탐색할 때 루프 카운터 인덱스가 정확히 `1+` 증가하는지, 그리고 전체 개수(`sslength`)를 초과하지 않고 정지 조건에 도달하는지 검증합니다.
  - **나쁜 예**: 정지 조건 없이 `(while (setq ent (ssname ss idx)) ...)` 로만 탐색.
  - **좋은 예**: `(while (< idx (sslength ss)) (setq ent (ssname ss idx)) ... (setq idx (1+ idx)))`.
- [ ] **나누기 0 방어 코드**: 기하학적 길이 계산이나 기울기를 나눌 때 나누는 값(분모)이 `0`이 아닌지 검증하십시오. 특히 두 점 좌표가 일치하는 경우를 주의합니다.
  - **예**: `(if (> (distance p1 p2) 1e-6) (/ 1.0 (distance p1 p2)))`
- [ ] **기하 병목 방어**: 평행한 두 선의 교차점을 계산하거나 영(0) 길이 선에 대해 오프셋을 가할 때 캐드가 뻗어버리지 않도록 사전에 유효성을 필터링합니다.

## ActiveX 및 COM 안정성

- [ ] **ActiveX 예외 포획**: VLA Offset, Boolean 연산, Intersection 등 고위험군 작업을 수행할 때는 CAD 내부 스레드를 보호하기 위해 `(vl-catch-all-apply 'vla-method (list args))`를 사용하여 예외를 안전하게 넘기도록 처리합니다.
- [ ] **객체 유효성 확인**: VLA 객체의 속성을 수정하거나 메서드를 지시하기 전에, 해당 객체가 도면 데이터베이스 상에서 지워지지 않고 유효하게 살아있는 상태인지 `(vlax-write-enabled-p obj)` 등을 호출하여 확인하십시오.
- [ ] **자원 해제**: 사용한 COM/ActiveX 객체 변수들은 `nil`로 덮어쓰거나 수동으로 해제하여 메모리 누수를 사전에 차단합니다.
